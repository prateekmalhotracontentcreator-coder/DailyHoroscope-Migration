## ✅ What Was Fixed This Session

### Fix 1 — Sign-in Lag (DONE ✅)
- **Problem:** Render free tier cold-starts caused 10-30s lag on sign-in after inactivity
- **Fix:** Added `window.addEventListener('focus', ping)` to `useKeepAlive.js` so backend wakes up when user returns to tab
- **Commit:** `f436f15`
- **Status:** Verified working — instant sign-in observed

### Fix 2 — Email Overflow on Home Page (DONE ✅)
- **Problem:** Long email IDs overflowed into adjacent columns on the Home page banner
- **Fix:** Added `truncate break-all` to email `<p>` in `UserAccountMenu.jsx`; added `min-w-0` to flex container in `Home.jsx`
- **Status:** Verified working

### Fix 3 — Panchang Frontend Routing (DONE ✅)
- **Problem:** NavBar links to `/panchang/today`, `/panchang/tithi`, `/panchang/choghadiya` etc. all showed blank pages
- **Root cause:** `PanchangPage.jsx` only handled `daily`, `calendar`, `festivals` — no ALIAS map
- **Fix:** Full replacement of `PanchangPage.jsx` with 6 views + ALIAS map
- **Commit:** `53f6f48`
- **Status:** Sub-nav renders correctly. Data loading blocked by backend (Fix 4 below)

### Fix 4 — Panchang Backend 500 Error (COMMITTED, AWAITING RENDER DEPLOY)
- **Problem:** `/api/panchang/daily` returning 500 Internal Server Error
- **Root cause:** `pyswisseph 2.10.x` is strict about integer types. `swe.CALC_RISE`, `swe.BIT_DISC_CENTER`, `swe.SUN` etc. return float-like objects that must be explicitly cast to `int()` before use
- **Error progression:**
  1. `TypeError: tuple cannot be interpreted as integer` — geo passed in wrong position
  2. `TypeError: must be real number, not tuple` — geo tuple not unpacked
  3. `TypeError: float object cannot be interpreted as integer` — swe constants not cast to int
- **Final fix:** Cast all swe constants with `int()`, unpack geo as individual `float` args
- **Commit:** `3e1c4bf` (pushed at 00:30 UTC)
- **Status:** ⏳ Render deploying (~2-3 min). Verify before starting new work.

## 🚨 CRITICAL: GitHub Push Protocol

**Always use the GitHub MCP connector. Never use the browser editor.**

```
# Read file + get SHA
github:get_file_contents(owner, repo, path)

# Push single file (use SHA from above)
github:create_or_update_file(owner, repo, path, content, message, branch, sha)

# Push multiple files in one commit
github:push_files(owner, repo, branch, files[], message)
```

Full protocol documented in `.claude/WORKING_PROTOCOL.md`
